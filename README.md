# Dynmap Discord Deployment Bot

A Discord bot that automatically deploys **Dynmap static website snapshots** to
GitHub Pages whenever a `.zip` file is uploaded to a monitored channel.

The expected ZIP contents match the `plugins/dynmap/web/` folder generated by
the Dynmap Minecraft plugin:

```
index.html
standalone/
tiles/
js/
css/
...
```

All files are extracted directly into the root of your GitHub Pages repository.

---

## Quick Start

### 1 — Clone or download this bot

```bash
git clone https://github.com/yourname/dynmap-discord-bot
cd dynmap-discord-bot
```

### 2 — Install Python dependencies

Python 3.11+ is recommended.

```bash
pip install -r requirements.txt
```

### 3 — Create a Discord Application & Bot

1. Go to <https://discord.com/developers/applications> and click **New Application**.
2. Open the **Bot** tab → click **Add Bot**.
3. Under **Privileged Gateway Intents**, enable **Message Content Intent**.
4. Copy the **Token** — you will need it for `DISCORD_TOKEN`.
5. Go to **OAuth2 → URL Generator**, select scopes `bot` and permissions
   `Read Messages/View Channels`, `Send Messages`, `Read Message History`.
6. Open the generated URL in your browser and invite the bot to your server.

### 4 — Prepare your GitHub Pages repository

```bash
# Clone your GitHub Pages repo to the machine that will run the bot
git clone https://github.com/yourname/your-pages-repo /home/user/my-dynmap-pages

# Make sure git push works without a password prompt by configuring
# SSH keys or a HTTPS credential helper, e.g.:
git remote set-url origin git@github.com:yourname/your-pages-repo.git
```

> **GitHub Pages configuration**
>
> In your repository settings → *Pages*, set the source to the **main** (or
> **master**) branch and the folder to **/ (root)**.
> GitHub will automatically publish every push to that branch.

### 5 — Configure environment variables

```bash
cp .env.example .env
# Edit .env with your values
```

| Variable | Required | Description |
|---|---|---|
| `DISCORD_TOKEN` | ✅ | Bot token from the Discord Developer Portal |
| `CHANNEL_ID` | ✅ | Numeric ID of the channel to monitor |
| `GITHUB_REPO_PATH` | ✅ | Absolute path to the local GitHub Pages repo clone |
| `GITHUB_PAGES_URL` | ✅ | Public URL shown in success messages |
| `IGNORE_DUPLICATES` | — | `true` to skip re-deploying an identical ZIP |
| `DELETE_ZIP` | — | `true` to note deletion of temporary download |
| `GIT_USER_NAME` | — | Git commit author name (default: `Dynmap Bot`) |
| `GIT_USER_EMAIL` | — | Git commit author e-mail (default: `bot@dynmap`) |

### 6 — Run the bot

```bash
python bot.py
```

## How It Works

```
Discord upload → bot downloads ZIP in memory
              → cleans repo dir (preserves .git)
              → safely extracts ZIP to repo root
              → git add --all && git commit && git push
              → sends success/error message in Discord
```

**Path traversal protection** — every member path is validated to ensure
extraction stays inside the repository root before any file is written.

**Large tile set support** — files are streamed in 4 MiB chunks, so even
repositories with tens of thousands of Dynmap tile images are handled
efficiently without loading everything into memory at once.

## Running as a System Service (Linux)

Create `/etc/systemd/system/dynmap-bot.service`:

```ini
[Unit]
Description=Dynmap Discord Deployment Bot
After=network.target

[Service]
User=youruser
WorkingDirectory=/path/to/dynmap-discord-bot
EnvironmentFile=/path/to/dynmap-discord-bot/.env
ExecStart=/usr/bin/python3 bot.py
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now dynmap-bot
sudo journalctl -u dynmap-bot -f   # follow logs
```

## Troubleshooting

| Symptom | Fix |
|---|---|
| Bot does not respond to uploads | Confirm **Message Content Intent** is enabled in the Developer Portal and `CHANNEL_ID` is correct |
| `Permission denied` when pushing | Ensure SSH keys are set up or use a HTTPS token in the remote URL |
| `git push` prompts for password | Switch the remote to SSH or configure a credential helper (`git config credential.helper store`) |
| ZIP extracted to wrong location | The ZIP must contain Dynmap web files at its root (not inside a subfolder). Re-export from `plugins/dynmap/web/` without an extra wrapper folder |
| `DISCORD_TOKEN` not found | Make sure `.env` exists in the same directory as `bot.py` or export the variable in your shell/service |

## Dynmap ZIP Export Tips

On your Minecraft server, the Dynmap web directory is usually at:

```
plugins/dynmap/web/
```

To create a deployable ZIP:

```bash
cd plugins/dynmap/web
zip -r /tmp/dynmap-snapshot.zip .
```

Then upload `/tmp/dynmap-snapshot.zip` to your monitored Discord channel.

The bot will place `index.html`, `tiles/`, `standalone/`, `js/`, `css/`, and
all other files directly in your GitHub Pages repository root.
